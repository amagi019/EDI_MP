# 管理側 設計書

## 1. 機能概要
自社管理者向けの全体進捗把握および履歴管理機能。

- **進捗管理ダッシュボード**: 帳票のDL状況や未確認の取引先を一目で把握。
- **履歴管理**: 誰が、いつ、どの帳票をDLしたかの証跡を保持。

## 2. 画面設計
- **ダッシュボード画面**: 
  - 月別のステータス集計グラフ（未受領/受領済）。
  - 未受領取引先の一覧（アラート表示）。
  - 最新のDLアクティビティログ。

## 3. 入出力仕様
### 出力項目
- 取引先ごとの受領完了率。
- 帳票DL履歴一覧（日時、取引先名、帳票ID、操作ユーザー）。

## 4. 業務ロジック
- **未確認リスト抽出**: `Order` の `status` が 'RECEIVED' 以外、かつ `order_date` から一定期間経過しているものを抽出。
- **履歴追跡**: `DownloadLog` （必要に応じて新規追加）等のログテーブルにDLイベントを記録。

## 5. エラーハンドリング
- データ集計時のパフォーマンス低下（キャッシュ利用を検討）。

## 6. セキュリティ要件
- 管理者権限を持つユーザーのみがアクセス可能。
- 操作ログの改ざん防止。
