# 管理者向け操作マニュアル

EDIシステムの管理者（スタッフ）向け操作マニュアルです。  
月次の発注業務フローと、イレギュラー対応手順を記載しています。

---

## 1. 月次業務フロー（正常系）

毎月、以下の流れで発注〜支払を管理します。

```
前月中旬        前月下旬        当月末        翌月上旬        翌々月15日
  ↓              ↓             ↓             ↓              ↓
注文書作成 → 注文書承諾 → 稼働報告受領 → 請求書作成 → 支払完了
(STEP 1)    (STEP 2)     (STEP 3)     (STEP 4-5)    (STEP 6-7)
```

### STEP 1: 注文書作成（前月中旬）

1. サイドバー「📋基本発注情報」を開く
2. 対象パートナーの行にある「📄注文書作成」ボタンをクリック
3. 翌月分の注文書ドラフトが自動生成される
4. 内容を確認し、「正式発行」ボタンをクリック
   - PDFが生成され、パートナーにメールで通知される
   - Google Driveに自動保存される

> **ポイント**: 基本発注情報が正しく設定されていれば、金額・支払条件・担当者は全て自動入力されます。

### STEP 2: 注文書承諾の確認（前月下旬）

- パートナーがシステムにログインし「承諾」ボタンを押すと完了
- ダッシュボードのパイプラインで「注文承認」ステップが緑になる
- **2日以上承諾されない場合**: 毎朝のチェックバッチで警告メールが届く

### STEP 3: 稼働報告書の受領（当月末）

- パートナーが月末にExcelファイルをアップロード
- ダッシュボードの「報告受領」ステップが緑になる
- **月末3営業日前**: パートナーにリマインドメールが自動送信される

### STEP 4: 請求書の作成（翌月上旬）

1. ダッシュボードで対象行をクリック → 「📝注文書を見る」
2. 注文書詳細画面の下部「請求書作成」ボタンをクリック
3. 請求書が自動生成される（金額は注文書から転記）

### STEP 5: 請求書の送付

- 請求書のステータスを確認し、パートナーに送付

### STEP 6-7: 支払完了

1. 振込完了後、ダッシュボードで対象行をクリック
2. 「💰支払完了」ボタンをクリック
3. パイプラインが全て緑（完了）になる

## 2. ダッシュボードの操作

### パイプラインの見方

| 色 | 意味 |
|----|------|
| 🟢 緑 | 完了 |
| 🟡 黄/オレンジ | 進行中（期限前） |
| 🔴 赤 | 期限超過 |
| ⬜ グレー | 未着手 |

### 行をクリック → 詳細パネル

行をクリックすると、下部に詳細パネルが開きます。以下のボタンが使えます：

| ボタン | 機能 |
|--------|------|
| 📋月次タスク完了 | 未完了タスクを一括で完了にする |
| 💰支払完了 | 支払ステップを完了にする |
| 📝注文書を見る | 注文書詳細画面に遷移する |
| ✕閉じる | 詳細パネルを閉じる |

### フィルタ

ヘッダーの切り替えボタンで表示を絞り込めます：
- **未完了**: 完了していない行のみ（デフォルト）
- **遅延**: 期限超過のタスクがある行のみ
- **完了**: 全ステップ完了した行のみ
- **全て**: 全行表示

## 3. イレギュラー対応

### 注文書の訂正再送付

発行済み注文書に誤りがあった場合（支払条件の間違い等）：

1. ダッシュボードで対象行をクリック → 「📝注文書を見る」
2. 注文書詳細画面で内容を確認
3. **修正が必要な場合**:
   - 「編集」ボタンで内容を修正（DRAFTの場合のみ）
   - または管理画面（/admin/）から直接データを修正
4. 「訂正再送付」ボタン（赤枠）をクリック
5. 確認ダイアログで「OK」
6. パートナーに訂正版PDFとお詫びメールが自動送信される

> **メール件名**: 【訂正】注文書 MPXXXXXXXXX の訂正版送付のお詫びとご連絡

### タスクの手動完了

パイプラインのステップが実際には完了しているのに未完了のままの場合：

1. ダッシュボードで対象行をクリック
2. 「📋月次タスク完了」ボタンで一括完了
3. または、個別ステップの「→」ボタンでスキップ

### 支払完了の手動マーク

請求書がない場合（過去月データの補完など）でも支払完了にできます：

1. ダッシュボードで対象行をクリック
2. 「💰支払完了」ボタンをクリック

### 基本発注情報の変更

契約期間・金額・支払条件が変更になった場合：

1. サイドバー「📋基本発注情報」を開く
2. 対象パートナーの「編集」ボタンをクリック
3. 変更内容を入力して保存
4. **既に発行済みの注文書**には反映されないため、訂正再送付が必要な場合は個別に対応

## 4. 支払条件の管理

支払条件（PaymentTerm）は構造化データとして管理されています。

### 設定項目

| 項目 | 説明 | 例 |
|------|------|-----|
| 締め日 | 0=月末, 15=15日締め | 0（月末） |
| 支払月オフセット | 締め日から何ヶ月後か | 2（翌々月） |
| 支払日 | 0=月末, 15=15日 | 15（15日払い） |

**例**: 月末締め・翌々月15日払い → 締め日=0, オフセット=2, 支払日=15

### 支払期限の自動計算

- 請求書作成時に、注文書に紐付いたPaymentTermから自動計算
- 例: 4月分作業 → 月末締め → 翌々月(6月)15日 → 支払期限 2026/06/15

## 5. 警告メール

毎朝9:00に自動チェックが実行され、以下の場合に管理者へ警告メールが送信されます：

| チェック項目 | 条件 |
|-------------|------|
| 注文書未承諾 | 発行から2日以上経過 |
| 支払期限間近 | 期限まで7日以内 |
| 支払期限超過 | 期限を過ぎている |

### 手動実行

```bash
# ドライラン（送信せず確認のみ）
docker compose -f docker-compose.nas.yml exec -T web python manage.py check_overdue_tasks --dry-run

# 実行
docker compose -f docker-compose.nas.yml exec -T web python manage.py check_overdue_tasks
```

注文書・タスク・請求書の操作ログは `/app/logs/audit.log` に30日分保持されます。

## 7. よくある質問

### Q: 注文書が承諾されない

- パートナーにメールが届いているか確認
- パートナーがシステムにログインできるか確認
- 必要に応じてパートナーに直接連絡

### Q: パイプラインの表示がおかしい

- ブラウザをリロード（Ctrl+R / Cmd+R）
- キャッシュクリア（Ctrl+Shift+R / Cmd+Shift+R）
- それでも解決しない場合はデータの整合性を確認

### Q: 過去月のデータを修正したい

- 管理画面（/admin/）から直接データを修正可能
- OrderCycle、MonthlyTask、Order のデータを確認
