# 請求管理 設計書

## 1. 機能概要
取引先に対する月次の検収金額を確定し、取引先が請求（支払）明細を取得できる機能。

- **自社作成請求書**: 自社で計算した金額（基本給＋超過精算等）を確定し、EDIに公開。
- **請求データDL**: 取引先がインボイス対応のPDFやCSVデータを取得。

## 2. 画面設計
- **請求明細一覧画面**: 月ごとの請求額一覧。
- **請求明細詳細画面**: 内訳（作業時間、精算額、源泉徴収、消費税等）の表示。

## 3. 入出力仕様
### 出力項目
- 支払通知書PDF（インボイス制度対応フォーマット、登録番号等）
- 請求CSV（日付、項目、数量、単価、金額、消費税区分）

## 4. 業務ロジック
- **請求金額計算**: `Order` の `base_fee` と稼働時間（外部データ想定）に基づき、`over_limit` / `shortage_fee` に応じた精算額を算出。
- **ステータス管理**: 作成中、公開済。公開後に取引先への自動通知を行う。

## 5. エラーハンドリング
- 計算元データの不足。
- 承認前の公開制限。

## 6. セキュリティ要件
- インボイス制度要件に基づくデータ改ざん防止措置。
