# テストシナリオ：パートナー登録から基本契約締結まで

## 概要
本シナリオは、新規パートナー企業とのEDI取引開始に必要な「パートナー登録→基本契約締結」までの一連のステップを、誰が・どこで・何を・どうするかを一つずつ記載したテスト手順書です。

---

## 前提条件
- 管理者アカウント（admin / admin123）でログイン可能であること
- SMTPメール設定が完了していること
- Googleドキュメントの電子署名機能が利用可能であること（手動工程）

---

## ステップ一覧

### ステップ1: 自社担当者がパートナーを新規登録する

| 項目 | 内容 |
|:---|:---|
| **操作者** | 自社担当者 |
| **場所** | EDIシステム → ダッシュボード |
| **操作** | 1. `http://127.0.0.1:8090/` にアクセスし、管理者アカウントでログインする |
| | 2. ダッシュボードの **「パートナー新規登録」** ボタンをクリックする |
| | 3. パートナー登録フォームに以下を入力する: |
| | 　- 会社名（例: 株式会社テストパートナー） |
| | 　- 担当者のメールアドレス |
| | 　- 担当者のログイン用ユーザー名 |
| | 4. **「パートナーとユーザーを登録する」** ボタンをクリックする |
| **期待結果** | ・登録完了画面が表示され、ログインID・初期パスワードが表示される |
| | ・基本契約進捗が **「招待済み（INVITED）」** のステータスで自動作成される |

---

### ステップ1.5: 自社担当者が招待メールをプレビュー確認し送信する

| 項目 | 内容 |
|:---|:---|
| **操作者** | 自社担当者 |
| **場所** | EDIシステム → Django管理画面 → Partners |
| **操作** | 1. Django管理画面（`/admin/`）にアクセスする |
| | 2. **Core → Partners** を開く |
| | 3. 登録したパートナーにチェックを入れる |
| | 4. アクション欄から **「招待メールを作成」** を選択し、「Go」をクリックする |
| | 5. メールプレビュー画面が表示されるので、**件名・本文を確認する** |
| | 6. 内容に問題がなければ **「送信する」** ボタンをクリックする |
| **期待結果** | ・パートナーの担当者メールアドレスに招待メール（ログインURL・ID・パスワード）が送信される |
| | ・メールログに送信記録が保存される |

---

### ステップ2: 自社担当者が基本契約進捗を確認する

| 項目 | 内容 |
|:---|:---|
| **操作者** | 自社担当者 |
| **場所** | EDIシステム → ダッシュボード → 基本契約進捗 |
| **操作** | 1. ダッシュボードの **「基本契約進捗」** ボタンをクリックする |
| | 2. 新規登録したパートナーが一覧に表示されていることを確認する |
| | 3. ステータスバッジが **「招待済み」**（紫色）であることを確認する |
| | 4. **「詳細」** ボタンをクリックしてメールログ画面を開く |
| **期待結果** | ・パートナーメールログ画面がエラーなく表示される |
| | ・招待メールの送信記録が一覧に表示される |

---

### ステップ3: パートナーが招待メールを受信しログインする

| 項目 | 内容 |
|:---|:---|
| **操作者** | パートナー担当者 |
| **場所** | メールクライアント → EDIシステム ログイン画面 |
| **操作** | 1. 招待メールを確認し、記載されたEDIシステムのURL・ログイン情報を確認する |
| | 2. EDIシステム（`http://127.0.0.1:8090/accounts/login/`）にアクセスする |
| | 3. ステップ1で発行されたユーザー名・初期パスワードでログインする |
| **期待結果** | ・正常にログインでき、パートナー用ダッシュボードが表示される |

---

### ステップ4: パートナーがオンボーディング（会社情報登録）を行う

| 項目 | 内容 |
|:---|:---|
| **操作者** | パートナー担当者 |
| **場所** | EDIシステム → オンボーディング画面 |
| **操作** | 1. ダッシュボードまたはナビゲーションから **オンボーディング画面** にアクセスする |
| | 　URL: `/partner/onboarding/` |
| | 2. 以下の会社情報を入力する: |
| | 　- 会社の正式名称、代表者情報、住所、連絡先 |
| | 　- **インボイス登録番号**（T番号） |
| | 　- **振込先口座情報**（銀行名、支店名、口座番号、口座名義） |
| | 3. 必要に応じて決算報告書等の添付書類をアップロードする |
| | 4. 入力内容を確認し、**「送信」** ボタンをクリックする |
| **期待結果** | ・入力した情報がDBに保存される |
| | ・基本契約進捗のステータスが **「情報登録済（INFO_DONE）」** に自動更新される |
| | ・管理者に情報登録完了の通知が行われる |

---

### ステップ5: 管理者が情報登録完了を確認する

| 項目 | 内容 |
|:---|:---|
| **操作者** | 自社担当者 |
| **場所** | EDIシステム → 基本契約進捗 |
| **操作** | 1. ダッシュボード →  **「基本契約進捗」** をクリックする |
| | 2. 該当パートナーのステータスが **「情報登録済」**（黄色）に変わっていることを確認する |
| | 3. Django管理画面（`/admin/`）→ Partners からパートナー情報が正しく登録されているか確認する |
| **期待結果** | ・ステータスが「情報登録済」であること |
| | ・パートナーの会社情報・口座情報・添付書類が正しく保存されていること |

---

### ステップ6: 自社担当者がEDIで基本契約書を作成する

| 項目 | 内容 |
|:---|:---|
| **操作者** | 自社担当者 |
| **場所** | EDIシステム → 基本契約進捗一覧 |
| **操作** | 1. ダッシュボード → **「基本契約進捗」** をクリックする |
| | 2. 該当パートナーのステータスが「情報登録済」であることを確認する |
| | 3. **「契約書作成」** ボタンをクリックする |
| **期待結果** | ・システムがパートナー情報（社名・住所・代表者名）をDBから取得し、契約書PDFを自動生成する |
| | ・PDFにSHA256ハッシュ値が記録される（電帳法対応） |
| | ・ステータスが **「基本契約送信済み（CONTRACT_SENT）」** に遷移する |
| | ・甲（自社）の角印が契約書に表示されること |

---

### ステップ7: 自社担当者が契約書をプレビュー確認し送信する

| 項目 | 内容 |
|:---|:---|
| **操作者** | 自社担当者 |
| **場所** | EDIシステム → 基本契約進捗一覧 |
| **操作** | 1. **「契約書PDF」** ボタンをクリックしてプレビューを確認する |
| | 2. 内容に問題がなければ **「送信」** ボタンをクリックする |
| | 3. 確認ダイアログで「OK」をクリックする |
| **期待結果** | ・パートナーの担当者メールアドレスに契約書確認依頼メールが送信される |
| | ・メール本文に契約書確認URL（EDI上の承認画面）が記載されていること |
| | ・メールログに送信記録が保存される |
| | ・ステータスが **「承認待ち（PENDING_APPROVAL）」** に遷移する |

---

### ステップ8: パートナーがEDIで契約書を確認し承認する

| 項目 | 内容 |
|:---|:---|
| **操作者** | パートナー担当者 |
| **場所** | EDIシステム → 契約書承認画面 |
| **操作** | 1. メールに記載された契約書確認URLにアクセスする（またはEDIにログイン後、承認画面に遷移する） |
| | 2. **「契約書PDFを表示する」** をクリックし、契約内容を確認する |
| | 3. 内容に問題がなければ **「承認する」** ボタンをクリックする |
| | 4. 確認ダイアログで「OK」をクリックする |
| **期待結果** | ・システムが承認日時・承認者を記録する |
| | ・承認日時入りの契約書PDFが再生成される（乙の署名欄に「電子承認済」と承認日時が記載される） |
| | ・新しいPDFのSHA256ハッシュ値が更新される |
| | ・ステータスが **「締結完了（COMPLETED）」** に自動遷移する |
| | ・自社担当者に承認通知メールが送信される |

---

### ステップ9: 自社担当者が締結完了を確認する

| 項目 | 内容 |
|:---|:---|
| **操作者** | 自社担当者 |
| **場所** | EDIシステム → 基本契約進捗一覧 |
| **操作** | 1. ダッシュボード → **「基本契約進捗」** をクリックする |
| | 2. 該当パートナーのステータスが **「締結完了」**（緑色）であることを確認する |
| | 3. **「契約書PDF」** をクリックし、承認日時入りの最終版PDFを確認する |
| **期待結果** | ・ステータスバッジが「締結完了」（緑色）で表示される |
| | ・契約書PDFの乙欄に「電子承認済」と承認日時が記載されていること |
| | ・以降、このパートナーに対して注文書の発行が可能になる |

---

## 完了条件

上記ステップ1〜9が全てエラーなく完了し、以下が確認できること：

- [ ] パートナーが正常に登録されている
- [ ] 招待メールがプレビュー確認後に送信されている
- [ ] パートナーがログインできる
- [ ] オンボーディングで会社情報が登録されている
- [ ] 基本契約書PDFが自動生成されている
- [ ] 基本契約進捗が「招待済み」→「情報登録済」→「基本契約送信済み」→「承認待ち」→「締結完了」と正しく遷移している
- [ ] 承認済み契約書に電子承認日時が記録されている
- [ ] 基本契約締結後、注文書の発行フローに進める状態になっている
