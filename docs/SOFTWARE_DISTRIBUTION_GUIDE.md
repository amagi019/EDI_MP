# EDIシステム 他社導入・提供ガイドライン（管理者・運用者向け）

本システム（EDI・注文管理システム）を知人や他社等へパッケージとして提供・導入する際の手順書です。本システムはDockerを利用しているため、比較的簡単に別サーバー（NASやVPS等）へコピーして動作させることが可能です。

---

## 1. 事前準備（提供先に用意してもらうもの）

システムを動かすためには以下のインフラとアカウントが必要です。事前に先方へお伝えください。

1. **ホストサーバー（Docker環境）**: 
   - Linuxサーバー（Ubuntu等）、またはSynologyなどのNAS（Container Managerが動くモデル）
   - 最低スペック: メモリ2GB以上 (推奨4GB以上)
2. **Googleアカウント（Workspaceまたは無償版）**:
   - Google Drive連携（注文書・稼働報告書の自動保存）に使用します。
   - ※Google Cloud Consoleにて「Service Account（サービスアカウント）」キーの発行作業が必要です。
3. **送信用メールアカウント（SMTP情報）**:
   - システムから送られる自動通知メール（注文発行、稼働報告等の通知）の送信元になるメールサーバー情報（Gmail, SendGrid, または自社SMTPなど）。
4. **ドメイン・SSL証明書**（インターネット上で運用する場合）:
   - 先方社内のLANで完結する場合は不要ですが、社外（外注先）もアクセスするため、本来は必須です（例: `edi.other-company.com`）。

---

## 2. 導入ステップ（提供側の作業）

### ステップ①：ソースコードの複製（クローン）
本システムのリポジトリから、提供用（空っぽ）のコピーを作成します。
ソースコードの入ったフォルダ（例: `EDI_MP`）一式を先方のサーバーへ送付（SFTP等）します。
※その際、過去のデータベースファイル（`db.sqlite3`等）や自分の `.env` 設定ファイルが混入しないよう注意してください。

### ステップ②：設定ファイル (`.env`) の構築
サーバーにコードを配置したら、システムの「脳」となる設定ファイルを作ります。

```bash
cd EDI_MP
cp .env.example .env.nas
```

`.env.nas` ファイルを開き、以下の「先方固有の情報」に書き換えます。

- `SECRET_KEY`: セキュリティの鍵。ランダムな50文字程度の英数字に変更します。
- `ALLOWED_HOSTS`: 先方のIPアドレスやドメイン（例: `192.168.1.100,edi.company.com`）。
- **[データベース設定]**:
  - `POSTGRES_USER`
  - `POSTGRES_PASSWORD` (強固なパスワードに変更)
- **[メール設定]**:
  - `EMAIL_HOST_USER` (先方のアドレス)
  - `EMAIL_HOST_PASSWORD` (先方のアプリパスワード)
- **[Google DriveフォルダIDの設定]**:
  - `GOOGLE_DRIVE_CONTRACT_FOLDER_ID`: (契約書用)
  - `GOOGLE_DRIVE_ORDER_FOLDER_ID`: (注文書用)
  - `GOOGLE_DRIVE_WORK_REPORT_FOLDER_ID`: (稼働報告用)

### ステップ③：Googleサービスアカウントキーの設置
先方のGoogle Cloudで発行した `credentials.json` をシステムのルート直下に配置し、`.env.nas` にそのパス (`GOOGLE_DRIVE_CREDENTIALS_FILE=credentials.json`) を指定します。
※**重要**: 指定したGoogle Driveのフォルダに対して、このサービスアカウントのメールアドレス（`xxx@xxx.iam.gserviceaccount.com`）を「編集者」として共有（招待）しておく必要があります。

### ステップ④：Dockerコンテナの立ち上げ
設定が完了したら、以下のコマンドでシステム一式を起動させます。

```bash
# Docker ComposeでWebアプリとデータベースを同時に起動
docker compose -f docker-compose.nas.yml up -d
```

### ステップ⑤：初期マスターデータの作成とログイン
システムが立ち上がったら、先方の管理者用アカウント（スーパーユーザー）を作成します。

```bash
# コンテナの中に入ってコマンドを実行
docker compose -f docker-compose.nas.yml exec web python manage.py createsuperuser
```
「ユーザー名」「メールアドレス」「パスワード」を聞かれるので、先方のシステム担当者用の設定を入力します。

ブラウザから `http://[サーバーのIPアドレス]:8090/` にアクセスし、先ほど作成したIDとパスワードでログインできれば、システムの譲渡・構築は完了です！

---

## 3. 引継ぎとマニュアルの配布

構築が終わりましたら、以下の「日常業務マニュアル」と「初期ログイン情報」をセットにして先方へ納品してください。

- **提供物1**: 本文書に付属する `DAILY_OPERATIONS_MANUAL.md`（必要であればPDF等にして提供）
- **提供物2**: 先ほど `createsuperuser` で発行した「初期管理者ID」と「初期パスワード」
- **提供物3**: システムのURL（例: `http://192.168.x.x:8090/`）

先方は必ず最初に「取引先（クライアント）」および「パートナー（外注先）」のマスター登録から業務を開始することになります。
